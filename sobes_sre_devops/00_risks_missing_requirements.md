# Анализ `risks` и `missing_requirements` по подготовке к собеседованиям SRE/DevOps

## Цель документа
Собрать реальные риски, зафиксировать недостающие темы и закрыть их в едином формате подготовки.

## Risks
1. Фрагментарность покрытия тем.
2. Смещение в сторону теории без устной отработки.
3. Разный стиль заголовков и структуры.
4. Недостаточная системность в design-подготовке.
5. Неполное покрытие security/compliance для DevSecOps ролей.
6. Риск переоценки готовности без явного трекера.

## Missing Requirements
1. Kubernetes Core.
2. Observability + Incident Response.
3. SRE/DevOps System Design.
4. Cloud patterns и архитектурные trade-offs.
5. Security/compliance baseline в DevSecOps.
6. Единый формат: теория + вопросы (с ответами) + кейсы + красные флаги.

## Единый формат темы
1. Теория.
2. Вопросы с ответами.
3. Практические кейсы.
4. Красные флаги.

## Мини-чеклист готовности
- Могу объяснить тему за 3-5 минут без чтения.
- Могу разобрать минимум 1-2 кейса по шагам.
- Могу назвать trade-offs выбранного решения.
- Знаю красные флаги и не допускаю их в ответе.
