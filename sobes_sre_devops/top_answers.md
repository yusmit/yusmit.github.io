# Ответы к вопросам из `top.md`

Ниже ответы на вопросы приоритета **9–6** (самые частые для первичного и технического скрининга).
Формат: коротко и «устно», чтобы можно было проговаривать за 30–90 секунд.

## Приоритет 9

### Что такое load average? Что показывает эта метрика? Почему load average состоит из трёх значений?
Load average показывает среднее число задач в состоянии `R` (running/runnable) и `D` (uninterruptible sleep, обычно I/O wait), ожидающих CPU/ресурсов.  
Три значения — усреднение за 1, 5 и 15 минут, чтобы видеть мгновенную, среднюю и долгую тенденцию.  
Оценивать нужно относительно числа CPU: например, LA=8 на 8 vCPU — нормальная загрузка, LA=16 на 8 vCPU — перегруз/очередь.

### Опиши архитектуру Kubernetes-кластера, из чего состоит?
Есть **control plane** и **worker nodes**.  
Control plane: `kube-apiserver`, `etcd`, `scheduler`, `controller-manager` (иногда `cloud-controller-manager`).  
Worker: `kubelet`, контейнерный runtime (containerd/CRI-O), `kube-proxy`, CNI-плагин.  
Пользователь отправляет манифест в API server, scheduler выбирает ноду, kubelet запускает pod, service/networking обеспечивают доступ.

## Приоритет 8

### Как работает OOM killer и для чего нужен?
Когда ядру Linux не хватает памяти и ситуацию нельзя разрулить reclaim/swap, запускается OOM killer.  
Он выбирает процесс-жертву по `oom_score`/`oom_score_adj` и завершает его, чтобы освободить память и не уронить всю систему.  
В Kubernetes обычно убивается контейнер, превысивший лимиты памяти, и pod может уйти в `OOMKilled`.

## Приоритет 7

### От кого по умолчанию запускается контейнер? Почему это плохо?
По умолчанию контейнер часто стартует от `root` внутри контейнера.  
Это риск: при уязвимости приложения или контейнера злоумышленник получает повышенные привилегии и шанс на escape/доступ к хосту.  
Практика: `USER` в Dockerfile, `runAsNonRoot`, `readOnlyRootFilesystem`, drop capabilities.

### Что такое Readiness, Liveness, Startup пробы, какое отличие?
- `startupProbe` — проверка старта «тяжелого» приложения; пока не пройдет, liveness/readiness не срабатывают.
- `livenessProbe` — «жив ли процесс»; не проходит -> контейнер перезапускается.
- `readinessProbe` — «готов ли обслуживать трафик»; не проходит -> pod убирают из endpoints Service.

## Приоритет 6

### Отличие hardlink от symlink
Hardlink — дополнительное имя того же inode (один и тот же файл на уровне inode). Не работает между ФС, обычно не делают на директории.  
Symlink — отдельный файл-указатель на путь. Может ссылаться на другой FS и даже на несуществующий путь (битая ссылка).

### При создании файла ошибка `no space left on device` — причины?
Не только «кончился диск». Возможные причины:
- закончилось место на FS;
- закончились inode;
- переполнен отдельный раздел (`/var`, `/tmp`, overlay/docker);
- квоты пользователя/проекта;
- удаленные, но открытые файлы держат место (`lsof +L1`).

### Чем отличается TCP от UDP?
TCP — соединение, надежная доставка, порядок, контроль потока/перегрузки, ретрансляции.  
UDP — без установления соединения, без гарантий доставки/порядка, меньше overhead, ниже задержка.  
TCP — когда важна надежность; UDP — когда важна скорость/допустима потеря.

### Как TCP устанавливает соединение?
Через 3-way handshake:
1. Клиент `SYN`.
2. Сервер `SYN-ACK`.
3. Клиент `ACK`.
После этого соединение установлено, согласованы начальные sequence numbers.

### Как TCP поддерживает соединение?
Поддержка идет за счет:
- ACK и sequence numbers;
- окна и flow control;
- congestion control;
- ретрансляций при потере;
- keepalive (опционально) для обнаружения «мертвого» peer.

### Чем симметричное шифрование отличается от асимметричного?
Симметричное: один общий ключ на шифрование/дешифрование (быстро, для больших данных).  
Асимметричное: пара public/private ключей (медленнее, удобно для обмена ключом/подписи).  
В TLS обычно комбинируют: асимметрика для обмена сессионным ключом, дальше симметрика для трафика.

### Как работает ICMP?
ICMP передает служебные сообщения сети IP: unreachable, time exceeded, fragmentation needed, echo request/reply (ping).  
Нужен для диагностики и сигналов об ошибках доставки, сам не транспортирует пользовательские данные как TCP/UDP.

### DaemonSet — зачем нужен?
`DaemonSet` гарантирует запуск pod на каждой (или выбранных) нодах.  
Обычно для node-level агентов: логирование, мониторинг, security/EDR, CNI-компоненты, storage-agents.

### Что такое Taints и Tolerations?
`Taint` ставится на ноду и «отталкивает» pod'ы.  
`Toleration` в pod разрешает ему быть запланированным на ноду с соответствующим taint.  
Нужно для изоляции: системные, GPU, sensitive workloads, dedicated ноды.

### Что такое Requests и Limits?
- `requests` — гарантируемые ресурсы для планирования (scheduler на них ориентируется).
- `limits` — верхний предел потребления.
CPU limit троттлит, memory limit при превышении ведет к OOMKill.

### Affinity / anti-affinity
`affinity` задает предпочтения/требования, где запускать pod (node affinity, pod affinity).  
`anti-affinity` — наоборот, разносит pod'ы по нодам/зонам для отказоустойчивости.

### Как в Docker реализована изоляция контейнеров? Какие средства Linux используются?
Основа — `namespaces` (pid, net, mnt, uts, ipc, user) + `cgroups` (лимиты/учет ресурсов).  
Дополнительно: capabilities, seccomp, AppArmor/SELinux, chroot/mount isolation, overlayfs.

### Как уменьшить размер Docker-образа?
- multi-stage builds;
- минимальные base images;
- меньше лишних пакетов/кэшей;
- объединение команд установки и очистка в том же слое;
- `.dockerignore`;
- копировать только нужные артефакты;
- фиксировать версии зависимостей.

### Какие команды порождают слои в Dockerfile?
Слои создают команды, изменяющие filesystem: `RUN`, `COPY`, `ADD`.  
`ENV`, `CMD`, `ENTRYPOINT`, `EXPOSE` в основном меняют metadata/конфиг, а не «тяжелые» FS-слои.

### Что за `<none>` образы в `docker images`?
Обычно dangling/untagged images:
- старые слои после ребилда;
- промежуточные образы;
- образы без тега после retag/удалений.  
Чистятся `docker image prune` (осторожно в production).

### Что такое узел / нода?
Нода — сервер (физический или виртуальный), входящий в кластер.  
В Kubernetes нода запускает pod'ы и имеет kubelet/runtime/kube-proxy.


## Приоритет 5

### Что такое системные вызовы? Зачем они нужны и как работают? Примеры 5–10
Системный вызов — это «вход» из user space в kernel space для операций, которые может делать только ядро (файлы, сеть, процессы, память).  
Примеры: `open`, `read`, `write`, `close`, `fork`, `execve`, `wait4`, `socket`, `connect`, `mmap`, `ioctl`.

### `df` показывает занято 20 Гб, а `du` — 20 Мб. Почему?
Чаще всего: большие файлы удалены, но еще открыты процессом.  
`du` не видит удаленный путь, а `df` видит занятые блоки до закрытия дескриптора.  
Проверка: `lsof +L1`.

### Что такое Docker и какие Linux-механизмы в основе?
Docker — платформа упаковки и запуска приложений в контейнерах.  
В основе: namespaces, cgroups, capabilities, seccomp/AppArmor/SELinux, overlayfs, network namespaces/bridge.

### Виртуализация vs эмуляция
Виртуализация запускает гостевую ОС почти на «родной» архитектуре CPU (быстрее).  
Эмуляция имитирует другую архитектуру программно (медленнее, но гибче).

### StatefulSet vs Deployment
`Deployment` — для stateless workload, взаимозаменяемые pod'ы.  
`StatefulSet` — стабильные имена pod (`-0`, `-1`), порядок запуска/остановки, обычно с отдельными PVC на pod.

### Deployment vs ReplicaSet
`ReplicaSet` только держит нужное число pod.  
`Deployment` управляет ReplicaSet-ами и дает rollout/rollback/strategies.

### Что такое RAID и основные типы
RAID — объединение дисков для отказоустойчивости/производительности.  
RAID0 — скорость, без отказоустойчивости; RAID1 — зеркалирование; RAID5 — parity + минимум 3 диска; RAID6 — двойной parity; RAID10 — зеркала + stripe.

### Почему `available` больше `free` в Linux
`free` — совсем неиспользуемая память.  
`available` включает память, которую можно быстро освободить (page cache, reclaimable slab), поэтому обычно больше.

### Как объяснить SSH: установка соединения и авторизация паролем/ключом
Клиент подключается к 22/TCP, согласуются алгоритмы, сервер присылает host key, клиент проверяет его.  
Далее шифрованный канал.  
Пароль: сервер проверяет пароль пользователя.  
Ключи: клиент доказывает владение private key, сервер сверяет с `authorized_keys`.

### Как работают сертификаты и подтверждается соединение
Сертификат связывает публичный ключ с именем сервиса и подписан CA.  
Клиент проверяет цепочку доверия, срок, SAN/CN, отзыв.  
После проверки устанавливается защищенный канал (TLS handshake).

### Разница `truncate`, `delete`, `drop`
`truncate` — быстро очищает таблицу целиком (обычно без построчного логирования как delete).  
`delete` — удаляет строки по условию (можно откатить, триггеры/логика).  
`drop` — удаляет сам объект (таблицу/БД/индекс).

### Что такое ReplicaSet
Контроллер Kubernetes, поддерживающий нужное количество одинаковых pod по selector.

### Что такое Deployment
Высокоуровневый контроллер для stateless приложений: rollout, rollback, стратегии обновления.

### Что такое Service
Стабильная точка доступа к группе pod (виртуальный IP + DNS + балансировка).

### Типы Service
`ClusterIP`, `NodePort`, `LoadBalancer`, `ExternalName`.

### Что такое Ingress
L7-маршрутизация HTTP/HTTPS в Kubernetes (host/path rules) через ingress-controller.

### Что такое Job
Ресурс для выполнения конечной задачи до успешного завершения.

### Что такое CronJob
Планировщик Job по cron-расписанию.

### Что означает `apiVersion`
Версия API-группы Kubernetes для конкретного ресурса; определяет схему и контроллерную логику.

### Что такое namespace
Логическая изоляция ресурсов внутри кластера (имена, RBAC, quota/policies).

### Что такое ConfigMap
Хранение неконфиденциальной конфигурации (ключ-значение), монтируется как env/файлы.

### Что такое PV и PVC
`PV` — ресурс хранилища в кластере.  
`PVC` — запрос хранилища от pod.  
PVC привязывается к подходящему PV.

### Что такое псевдофайловая система
FS, не хранящая данные на диске в обычном виде, а отдающая представления ядра/процессов (`/proc`, `/sys`, `tmpfs`).

### Что такое LVM и для чего
Логический слой управления дисками: `PV -> VG -> LV`.  
Нужен для гибкого изменения размеров, снапшотов, миграций и управления storage без жесткой привязки к одному диску.

### Контейнер падает с `/bin/bash not found` — как диагностировать
Проверить базовый образ и shell: возможно там только `/bin/sh` (alpine/distroless).  
Проверить `Dockerfile`, `ENTRYPOINT/CMD`, `docker inspect`, попробовать запуск с `/bin/sh`.  
Проверить архитектуру образа и наличие бинарника в слое.

### Ограничение CPU/RAM/IO/Network у контейнера — можно?
Да. Основа — cgroups.  
CPU/RAM: `--cpus`, `--memory`; в Kubernetes — requests/limits.  
IO: `blkio` настройки.  
Network: tc/qdisc или CNI/policy/egress shaping.

### Что такое слои в Docker
Образ состоит из read-only слоев; контейнер добавляет writable слой сверху (copy-on-write).

### ARG vs ENV
`ARG` — переменная этапа сборки (build-time), обычно не доступна в runtime.  
`ENV` — переменная окружения в образе/контейнере (runtime).

### Что такое роли в Ansible (пример)
Роль — переиспользуемый модуль структуры автоматизации (`tasks`, `handlers`, `templates`, `vars`, `defaults`).  
Пример: роль `nginx` устанавливает пакет, кладет конфиг из шаблона, перезапускает сервис через handler.

## Приоритет 4

### Можно ли ограничить видимость процессов только своими?
Да: включить `hidepid=2` для `/proc` (через `/etc/fstab`), плюс корректные группы/RBAC/pam.

### Где посмотреть сводку по памяти
`free -h`, `cat /proc/meminfo`, `vmstat`, `top/htop`.

### Что такое контейнер и образ
Образ — неизменяемый шаблон с приложением/зависимостями.  
Контейнер — запущенный экземпляр образа с изоляцией и writable слоем.

### Pod vs контейнер
Контейнер — один процессный runtime-юнит.  
Pod — минимальная единица в Kubernetes, может включать 1+ контейнеров с общими network namespace и volume.

### Что происходит при вводе `yandex.ru` в браузере
Проверка кеша/hosts -> DNS резолв -> TCP/TLS -> HTTP(S) запрос -> ответ сервера (возможны CDN/redirect) -> рендер HTML/CSS/JS и дополнительные запросы.

### Статусы процессов в Linux
Основные: `R` (running), `S` (sleep), `D` (uninterruptible sleep), `T` (stopped), `Z` (zombie), `I` (idle kernel thread).

### Чем опасны зомби-процессы
Сами CPU почти не едят, но занимают PID-слоты в таблице процессов.  
Много zombie может привести к исчерпанию PID и проблемам запуска новых процессов.

### Что такое файловый дескриптор
Это число-идентификатор открытого файла/сокета/pipe у процесса.  
Хранит ссылку на inode/объект, текущий offset, flags, mode.

### Как процессы взаимодействуют
Signals, pipes/FIFO, Unix/TCP sockets, shared memory, message queues, semaphores, RPC.

### Для чего сигналы и 5–10 популярных
Сигналы — асинхронные уведомления процессам.  
Частые: `SIGTERM`, `SIGKILL`, `SIGINT`, `SIGHUP`, `SIGQUIT`, `SIGSTOP`, `SIGCONT`, `SIGCHLD`, `SIGUSR1`, `SIGUSR2`.

### Что такое inode
Структура метаданных файла: права, владелец, размер, timestamps, pointers на блоки данных. Имя файла хранится отдельно в директории.

### Что такое рекурсивный DNS
Рекурсивный резолвер сам проходит цепочку запросов (root -> TLD -> authoritative) и возвращает клиенту готовый ответ, обычно с кешированием.

### Разница ADD и COPY
`COPY` просто копирует файлы.  
`ADD` умеет распаковывать локальные архивы и брать URL (что часто менее предсказуемо). Обычно предпочтительнее `COPY`.

### На каком уровне работает TCP
Транспортный уровень (L4 модели OSI).

### Что такое TLS и SSL
SSL — устаревший предшественник.  
TLS — современный протокол защищенного канала (шифрование, целостность, аутентификация).

### Почему нежелательно использовать `latest`
Непредсказуемость и отсутствие воспроизводимости: сегодня `latest` один, завтра другой.  
Лучше фиксировать теги/дигесты.

### Как работает DNS и зачем нужен
Преобразует доменные имена в IP и обратно (разные типы записей).  
Нужен, чтобы использовать удобные имена вместо IP и управлять маршрутизацией/доступностью через DNS-записи.

### Основные типы HTTP-запросов
`GET`, `POST`, `PUT`, `PATCH`, `DELETE`, `HEAD`, `OPTIONS`.

### Что такое идемпотентность
Операция идемпотентна, если повтор дает тот же итоговый результат.  
Примеры: `PUT`, `DELETE` (обычно идемпотентны), `POST` часто неидемпотентен.


## Приоритет 3

### Что делает команда `kill`?
Отправляет сигнал процессу по PID. По умолчанию — `SIGTERM` (15), принудительно — `SIGKILL` (9).

### Как посмотреть нагрузку на диски?
`iostat -x`, `iotop`, `pidstat -d`, `sar -d`, метрики `await`, `svctm`, `util`, `queue depth`.

### Процесс и тред: отличия
Процесс имеет собственное адресное пространство. Треды делят память процесса, но имеют свои стеки/регистры. Треды легче и быстрее переключаются.

### У `chmod` убрали право исполнения, что делать?
Если есть право на изменение файла/директории: вернуть `chmod +x file`. Если нет — через владельца/root/ACL.

### Что происходит при `ENTRYPOINT`?
`ENTRYPOINT` задает основной исполняемый процесс контейнера. Аргументы `docker run ...` дополняют его (если не shell-form).

### `CMD` vs `ENTRYPOINT`
`ENTRYPOINT` — «что запускать». `CMD` — дефолтные аргументы/команда, легко переопределяется при `docker run`.

### Для чего `docker-compose`?
Для запуска и управления многоконтейнерными приложениями локально/в тесте единым YAML (`up/down`, сети, volume, env).

### Что такое Kubernetes?
Система оркестрации контейнеров: деплой, масштабирование, self-healing, сервис-дискавери, rollout/rollback.

### Что такое Pod?
Минимальная deploy-единица Kubernetes: 1+ контейнеров с общими сетью/томами.

### Что такое Helm?
Пакетный менеджер Kubernetes. Chart = шаблоны + values + версии релизов.

### Userspace и kernelspace
Userspace — пользовательские процессы, ограниченные привилегии. Kernelspace — код ядра с полным доступом к железу.

### Физическая память
Реальная RAM устройства.

### Виртуальная память
Абстракция адресного пространства процесса (RAM + swap + mmap), изолирует процессы и упрощает управление памятью.

### Что такое индексы в БД?
Структуры для ускорения поиска/сортировок (обычно B-tree, hash). Ускоряют чтение, но замедляют запись и занимают место.

### Запуск файла: `no such file or directory`, почему?
Причины: неверный путь, нет интерпретатора из shebang, несовместимая архитектура ELF, отсутствуют динамические библиотеки.

### IP и маска подсети
IP — адрес узла. Маска определяет, какая часть — сеть, какая — хост (`/24`, `/16` и т.д.).

### Зачем VLAN?
Логическая сегментация L2 сети, изоляция трафика и упрощение управления без отдельной физики.

### Рекурсивный и нерекурсивный DNS-запрос
Рекурсивный: сервер обязан вернуть финальный ответ/ошибку. Нерекурсивный: возвращает то, что знает (в т.ч. referral).

### Что такое Docker squash?
Объединение слоев образа в меньшее число слоев для уменьшения размера/утечки истории сборки.

### `docker stop` vs `docker pause`
`stop` отправляет `SIGTERM`/`SIGKILL` и завершает контейнер. `pause` замораживает процессы (cgroup freezer), контейнер не завершен.

### Как убить контейнер изнутри?
Завершить PID 1 внутри контейнера (`kill 1`/`exit` процесса) — контейнер остановится.

### Best practices Dockerfile
Минимальный base, pinned версии, multi-stage, `.dockerignore`, non-root user, минимум слоев/кэша, no secrets в образе.

### Почему DNS использует UDP?
Меньше overhead и быстрее для коротких запросов. TCP включается при больших ответах, zone transfer, DNSSEC/DoT/DoH и fallback.

### Как работает `sudo`?
Проверяет правила в `sudoers`, может запросить пароль, запускает команду с привилегиями другого пользователя (обычно root).

### Как работает NAT?
Преобразует адреса/порты на границе сети. Нужен для выхода private сетей в интернет и экономии публичных IP.

### Какую проблему решает Kubernetes?
Сложность ручного управления множеством контейнеров/узлов: масштабирование, отказоустойчивость, релизы, сервис-дискавери.

### Средства Linux для software RAID
`mdadm` (mdraid), LVM RAID, ZFS/Btrfs (встроенные механизмы).

### На каком уровне OSI работает SSH?
Прикладной уровень (L7), поверх TCP (обычно порт 22).

### Что такое маска подсети?
Битовая маска, отделяющая network bits от host bits в IP-адресе.

## Приоритет 2

### Вывод `top`: что означает
- `load average` — очередь runnable/uninterruptible задач.
- `Tasks` — общее число процессов и статусы.
- `%Cpu(s)` — user/system/iowait/idle и др.
- `MiB Mem/Swap` — распределение RAM/swap.
- В таблице: `PID USER PR NI VIRT RES SHR S %CPU %MEM TIME+ COMMAND`.

### Что такое zombie-процесс? Как создать?
Завершившийся дочерний процесс, которого родитель еще не `wait`-нул. Создать: форкнуть child, завершить child, parent не делает `wait()`.

### Load average 900/900/900, но сервер почти норм
Смотреть не только LA: `%idle`, `run queue`, `iowait`, PSI, latency приложения. Высокий LA может быть из-за большого числа D-state задач (I/O/NFS), а не CPU bottleneck.

### Обработка сигналов. SIGTERM vs SIGKILL
`SIGTERM` можно перехватить и корректно завершиться. `SIGKILL` перехватить нельзя — мгновенное убийство ядром.

### `export VAR=...` vs `VAR=...` в bash
Без `export` переменная локальна для текущего shell. С `export` доступна дочерним процессам как env.

### Где в Linux хранится информация о процессах?
В `procfs`: `/proc/<pid>/...`.

### Какой сигнал при Ctrl+C?
`SIGINT`.

### Что такое Kata Containers?
Контейнеры с дополнительной VM-изоляцией: легковесные ВМ + контейнерный UX, сильнее безопасность, чуть больше overhead.

### Зачем система инициализации? Примеры
Запускает userspace после ядра, управляет сервисами/зависимостями. Примеры: `systemd`, `SysV init`, `OpenRC`, `runit`, `s6`.

### Почему latency до 8.8.8.8 часто низкий?
Anycast и хорошая сеть Google. Но «всегда» неверно: зависит от маршрутизации, канала, перегрузки, фильтров ICMP.

## Приоритет 1

### Преимущества IPMI перед KVM-over-IP
IPMI дает out-of-band управление: питание, сенсоры, Serial-over-LAN, события, удаленный доступ даже при падении ОС. KVM-over-IP в основном про видео/клавиатуру/мышь.

### systemd vs init: главное преимущество
`systemd` быстрее и управляемее: параллельный старт, зависимости unit'ов, журналирование, таймеры, cgroups-интеграция.

### Что такое iowait и почему появляется?
Доля времени CPU, когда ядро ждет завершения I/O. Причины: медленный диск/сеть/storage, очереди I/O, перегрузка БД.

### Почему `mysqldump` плох на большой активной БД? Альтернативы
`mysqldump` долго, нагружает сервер, может давать долгие блокировки/рассинхрон. Альтернативы: `xtrabackup`, `mysqlpump`, реплика + backup с реплики, snapshot (LVM/ZFS/Storage).

## Приоритет 0

### Сервер не отвечает, как получить доступ без ЦОДа?
Через out-of-band: IPMI/iDRAC/iLO, KVM-over-IP, Serial-over-LAN, rescue mode провайдера, консоль гипервизора.

### Что такое KVM (гипервизор)?
KVM — модуль ядра Linux, превращающий его в гипервизор 1 типа. Обычно работает с QEMU для эмуляции устройств.

### Что такое IPMI и подсистемы
Стандарт удаленного управления сервером: BMC, датчики (SDR), журнал событий (SEL), управление питанием, удаленная консоль/SOL.

### Как посмотреть железо сервера
`lscpu`, `lsblk`, `lspci`, `lsusb`, `dmidecode`, `ipmitool sdr`, `smartctl`, `hwinfo`.

### Модель CPU, ядра, инструкции
`lscpu`, `/proc/cpuinfo`.

### Тип RAM, плата, BIOS
`dmidecode -t memory -t baseboard -t bios`.

### Сенсоры температуры/напряжения/вентиляторов
`ipmitool sdr`, `sensors`.

### Сетевой адаптер и интерфейсы
`lspci`, `ethtool`, `ip -br a`, `ip -s link`.

### Подключенные USB/PCI
`lsusb`, `lspci`.

### Процесс загрузки Linux от кнопки питания
POST/UEFI -> bootloader (GRUB) -> ядро + initramfs -> монтирование root FS -> `init/systemd` -> сервисы/target.

### Что такое POSIX
Набор стандартов интерфейсов ОС (API/утилиты/shell), обеспечивающий переносимость ПО.

### Что такое уровни выполнения
Старый SysV механизм режимов работы (`runlevels 0..6`). В systemd аналог — targets (`multi-user.target`, `graphical.target`).

### Основные компоненты Linux-дистрибутива
Ядро, init/system manager, userspace утилиты, package manager, libc, shell, сервисы, FS hierarchy.

### BIOS vs UEFI
BIOS — legacy прошивка. UEFI — современная, с GPT/secure boot/расширенными возможностями загрузки.

### Что такое PXE
Загрузка по сети: DHCP + TFTP/HTTP + network boot image.

### Ядро, initramfs, загрузчик
Загрузчик поднимает ядро и initramfs. initramfs готовит минимальное окружение и root FS, затем стартует init.

### Как понять, используется ли systemd
`ps -p 1 -o comm=` (должно быть `systemd`), `systemctl` доступен.

### Что происходит с процессами при запуске команды в shell
Shell парсит команду -> fork -> (в child) execve -> выполнение -> wait в parent (если foreground) -> код возврата.

### Что значит `$@` в bash
Все позиционные параметры как отдельные аргументы (в кавычках `"$@"` сохраняет границы аргументов).

### Как вывести только STDERR, игнорируя STDOUT
`command 1>/dev/null`.

### Как посмотреть описание дескриптора и mtime файла
Дескрипторы: `/proc/<pid>/fd`, `lsof -p <pid>`. Время модификации: `stat file`.

### Где найти информацию о конкретном системном вызове
`man 2 <syscall>`.

### Что такое SIGCHLD
Сигнал родителю при завершении/остановке дочернего процесса.

### Почему в htop сводка памяти может быть неточной
Различия в трактовке cache/buffers/slab/shared, cgroup-контекстах, задержка обновления данных.

### `a=5; true | { true && a=10; }` чему равно `a`?
Останется `5`, потому что правый блок выполняется в subshell из-за пайпа.

### Что такое QEMU
Эмулятор/виртуализатор машин. В связке с KVM дает аппаратно-ускоренную виртуализацию.

### Что такое qemu-kvm
QEMU + KVM-ускорение.

### Что такое символьные устройства и базовые операции
Устройства, работающие потоком байт (`/dev/tty`, `/dev/null`). Операции: open/read/write/ioctl.

### Что такое major/minor numbers устройств
`major` — драйвер, `minor` — конкретный экземпляр устройства у драйвера.

### Что такое файловая система и зачем
Способ организации хранения данных/метаданных на носителе.

### Как создать ФС на блочном устройстве, какие параметры
`mkfs.ext4 /dev/sdX1` или `mkfs.xfs`. Параметры: label, block size, inode options, features.

### Где физически находятся inode
На диске в структурах файловой системы (inode table или динамически, зависит от FS).

### Где могут закончиться inode и к чему это ведет
На FS с фиксированным числом inode (например, ext4 при создании). Итог: нельзя создать новые файлы при наличии свободных GB.

### Какая ФС с динамическими inode
XFS (и ряд CoW FS).

### Где хранится информация об именах файлов/директорий
В directory entries (каталоги связывают имя с inode).

### Как монтируется ФС и как посмотреть mounts
`mount`, `findmnt`, `cat /proc/mounts`.

### Проблемы ФС/диска и восстановление
Bad blocks, corruption journal/metadata, I/O errors. Восстановление: `fsck`/`xfs_repair` (вне mounted rw, по правилам FS).

### ext4 или xfs?
ext4 — универсальна и проста. XFS — хороша для больших файлов/параллельной нагрузки, динамические inode. Выбор по профилю нагрузки и ops-практикам.

### Как понять, на каком диске раздел ФС и как идентифицировать без выключения
`lsblk -f`, `blkid`, `findmnt -no SOURCE`, `udevadm info`, `by-id` symlinks.

### Как узнать, какими процессами используется раздел
`lsof <mountpoint>`, `fuser -vm <mountpoint>`, анализ `/proc/*/fd`.

### Для чего `/etc/fstab`, `/etc/mtab`, `/etc/mdadm/mdadm.conf`
`fstab` — автоподключение FS. `mtab`/`/proc/mounts` — текущие монтирования. `mdadm.conf` — конфиг/сборка RAID.

### Что такое loop devices
Псевдоблочные устройства для файлов-образов (`losetup`), удобно для ISO/образов/тестов.

### Ошибка 502 от веб-сервера — как искать
Проверить upstream доступность и таймауты, логи nginx/app, сокеты/порты, healthchecks, ресурсы, DNS/сертификаты.

### Файл с нечитаемым содержимым — как понять формат
`file`, `strings`, `hexdump/xxd`, `stat`, проверка magic/signature.

### Программа не находит конфиг — как узнать путь поиска
`strace -e file -f`, посмотреть попытки `openat`, документацию/`--help`/env vars.

### БД упирается в диск, железо не добавить — что делать быстро
Тюнинг запросов/индексов, кеш, перенос hot данных в RAM/SSD-tier, reduce fsync pressure, batch writes, партиционирование, read replica, ограничение конкурентности тяжелых запросов.

### Что такое разделяемая память
IPC-механизм, где процессы используют общий участок памяти (`shm`, `mmap`) для быстрого обмена.

### Под нагрузкой Cassandra/ELK тормозят — куда смотреть
Disk IO latency, GC, heap/off-heap, compaction/merge, hot shards/partitions, network RTT, thread pools, throttling, saturation в storage.

### В каком виде хранятся образы, зачем слои, что такое overlayfs
Образы — набор read-only слоев + metadata. Слои дают переиспользование и быстрый pull/build. OverlayFS объединяет слои и writable upperdir.

### Почему в контейнере видны только его процессы
Изоляция PID namespace: процесс видит только namespace-процессы (если не hostPID).

### В `/var/lib/docker` как понять папку контейнера
Через `docker inspect` (`GraphDriver.Data`, container ID), сопоставление с каталогами overlay2/containers.

### Можно ли настроить сеть между двумя контейнерами
Да: пользовательская bridge network, overlay (swarm), host network, macvlan. Контейнеры в одной сети видят друг друга по DNS-именам.

### Можно ли сделать авто-перезапуск контейнера
Да, политика `--restart` (`always`, `unless-stopped`, `on-failure`) или оркестратор.

### Что такое ARP
Протокол сопоставления IPv4 адреса и MAC в локальном сегменте L2.

### Что такое SNI
Расширение TLS ClientHello, передает целевое имя хоста для выбора сертификата на одном IP.

### Что такое authority в DNS
Authoritative DNS — сервер, который хранит «истинные» записи зоны и отвечает за домен.

### Базовая HTTP-аутентификация
Клиент отправляет `Authorization: Basic base64(user:pass)`. Без TLS небезопасно.

### Принцип SSL/TLS-шифрования
Handshake, проверка сертификата, выработка сессионных ключей, затем симметричное шифрование трафика + MAC/AEAD.

### Стандартные коды HTTP
1xx инфо, 2xx успех (200/201), 3xx редирект (301/302), 4xx ошибка клиента (400/401/403/404), 5xx сервера (500/502/503/504).

### Для чего ad-hoc в Ansible
Быстрые разовые команды без playbook (`ansible all -m ping`).

### Для чего handlers в Ansible
Отложенные действия по событию изменения (например, перезапуск сервиса только при изменении конфига).

### Pull vs push модель
Push: контроллер отправляет изменения на узлы (Ansible). Pull: агент на узле сам забирает конфиг (Puppet/chef-client и т.д.).

### Плюсы Ansible
Простота, agentless (SSH), YAML, идемпотентность, быстрая автоматизация и оркестрация.

### Основные примитивы Ansible
Inventory, Playbook, Task, Module, Role, Handler, Variable, Template, Facts, Tags.

### Модуль vs плагин в Ansible
Модуль выполняет конкретное действие (например `copy`, `service`). Плагин расширяет поведение движка (connection, callback, filter, lookup).

### Базы MySQL — базово
Транзакции (InnoDB), индексы, блокировки, репликация, backup/restore.

### Проблема большого числа индексов
Ускоряют чтение, но замедляют запись/обновления, увеличивают размер БД и время обслуживания.

### Как настроить master-slave репликацию MySQL (кратко)
Включить binlog на мастере, создать repl-user, снять позицию/GTID, на реплике настроить источник и запустить replication threads.

### Что такое роли в PostgreSQL
Унифицированные сущности пользователей/групп для прав доступа, логина и управления привилегиями.

### Что такое хеш-таблица
Структура ключ-значение с хеш-функцией, средняя сложность доступа O(1).

### Генератор и итератор (Python)
Итератор реализует протокол `__iter__/__next__`. Генератор — удобный способ создать итератор через `yield`.

### Кортеж vs список
Tuple — неизменяемый, list — изменяемый.

### Что такое декоратор
Функция/объект, оборачивающая другую функцию/метод для добавления поведения без изменения исходного кода.

### Значение переменной без значения в Python
`None`.

### Что такое minikube
Локальный одноузловой Kubernetes для обучения/разработки.

### Пример проблемы, которую упрощает Kubernetes
Без простоя обновлять множество сервисов с rollback и автовосстановлением при падении pod.

### Что такое оператор в Kubernetes
Контроллер + CRD, который кодирует доменную операционную логику (day-2 ops) для приложения.

### Как создается pod, какие компоненты задействованы
Манифест -> API server -> etcd -> scheduler выбирает ноду -> kubelet вызывает runtime -> CNI/CSI подключают сеть/тома.

### Может ли pod запуститься на двух узлах одновременно
Конкретный pod — нет. Но реплики одного workload могут быть на разных узлах.

### Что такое Volume
Способ предоставить контейнеру хранилище (ephemeral или persistent), переживающее рестарт контейнера в pod.

### Типы файловых хранилищ (базово)
Блочные, файловые, объектные; в Kubernetes — local, NFS, CSI-backed cloud/storage classes.

### Что такое Secret
Kubernetes-объект для хранения чувствительных данных (base64, желательно с encryption at rest/внешним vault).

### Что такое nodeSelector, nodeName
`nodeSelector` — отбор ноды по label. `nodeName` — жестко указать конкретную ноду.

### Через что реализованы сети в Kubernetes
Через CNI-плагины (Calico/Cilium/Flannel и др.), kube-proxy/eBPF dataplane, Service abstraction.

### Что будет при изменении image? Как докатятся изменения
Deployment создаст новый ReplicaSet и выполнит rolling update по стратегии (`maxSurge/maxUnavailable`), readiness контролирует подачу трафика.

### Merge vs rebase
`merge` сохраняет историю ветвления и добавляет merge commit. `rebase` переписывает commits поверх новой базы, история линейнее.

### Когда merge, когда rebase
Merge — безопасно для публичных/shared веток. Rebase — для локальной очистки истории до публикации.

### `git pull` vs `git fetch`
`fetch` только скачивает обновления. `pull` = fetch + merge/rebase в текущую ветку.

### Что такое cherry-pick
Применение конкретного commit из другой ветки в текущую.

### Практики работы с Git
Feature branches, PR/MR, code review, protected branches, small commits, conventional messages, форки при внешнем контрибьюте.

### Что такое GitFlow
Модель ветвления с `main`/`develop`, feature/release/hotfix ветками. Полезна в ряде процессов, но часто упрощается под trunk-based.
